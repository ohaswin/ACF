# Database Models

The Autonomous Content Factory relies on three simple models inside `campaigns/models.py`. We use Django's robust built-in ORM to interface with our SQLite datastore.

## 1. `Campaign`
This is the root object. When a user pastes text, uploads a file, or enters a URL, the raw text is extracted and stored here immediately.

### Key Fields:
- `source_text (TextField)`: The raw text extracted from the user's input.
- `fact_sheet (JSONField)`: The structured dictionary returned by Agent 1. It holds the core facts, features, and target audience required by the writers.
- `status (CharField)`: Tracks the holistic progress (`pending` > `researching` > `writing` > `reviewing` > `completed`).

## 2. `ContentPiece`
A Campaign has many `ContentPiece`s (One-to-Many). These represent the actual drafted blocks of text produced by Agent 2 (The Copywriter). 

Whenever Agent 3 (The Editor) reviews the draft, it updates the `editor_feedback` and ticks the `iteration_count`.

### Key Fields:
- `campaign (ForeignKey)`: Linked to the parent `Campaign`.
- `channel (CharField)`: The target medium (`blog`, `social`, or `email`).
- `content (TextField)`: The raw markdown text generated by the LLM.
- `editor_feedback (TextField)`: If Agent 3 rejects the text, it dumps its reasoning here. This is then fed back to Agent 2.
- `iteration_count (IntegerField)`: How many times Agent 2 has attempted this channel. Defaults to 1.
- `status (CharField)`: The granular state of this specific channel (`draft` > `approved` or `rejected`).

## 3. `AgentLog`
The AI pipeline operates asynchronously via Celery. Instead of letting `print()` statements disappear into `/celery.log`, we save them to the database so the User Dashboard GUI can display a live, permanent history.

### Key Fields:
- `campaign (ForeignKey)`: The parent campaign.
- `agent_name (CharField)`: Identifier (e.g., "Agent 2 (Copywriter)" or "System").
- `message (TextField)`: The log message.
- `created_at (DateTimeField)`: When the log occurred. Ordered descending on the frontend to create a chat interface.

### The `log_agent_action()` utility
Whenever the background worker wants to say something, it calls this helper. This function simultaneously creates an `AgentLog` in the DB AND broadcasts the payload via Django Channels to the WebSocket listener.
